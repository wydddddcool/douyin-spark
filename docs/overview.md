# 项目概述

## 项目背景

抖音的"火花"是抖音好友之间连续聊天天数的象征。为了保持与好友的火花不熄灭，需要每天至少发送一条消息。这个项目就是为了解决这个问题而设计的自动化工具。

## 设计理念

1. **简单易用**: 提供友好的Web控制面板，无需技术背景的用户也能轻松使用
2. **安全可靠**: 采用本地存储登录凭证，不向第三方泄露
3. **灵活定制**: 支持自定义消息模板和定时配置
4. **反识别防护**: 使用真实浏览器环境和反检测技术
5. **跨平台**: 支持macOS、Windows和Linux

## 核心特性

### 1. 认证系统
- 扫码登录，无需输入账号密码
- 持久化存储登录状态（`state.json`）
- 支持多账号管理

### 2. 消息系统
- 每日不同风格的消息模板
- 支持自定义模板
- 变量替换（时段、日期、星期等）

### 3. 定时任务
- 灵活的定时配置
- 支持按星期几选择
- 开机自启动（macOS通过launchd）

### 4. Web控制面板
- 实时状态查看
- 扫码登录
- 好友管理
- 日志查看
- 消息预览

## 关键技术决策

### 为什么选择 Playwright？

- 相比 Selenium：
  - 更现代的API设计
  - 更好的浏览器自动化能力
  - 内置反检测机制
  - 更好的性能和稳定性

### 为什么选择系统 Chrome 而非内置浏览器？

- 抖音对自动化浏览器检测严格
- 真实浏览器指纹更难被识别
- 打包后无需内置庞大的浏览器二进制文件

### 为什么使用 YAML 配置？

- 人类可读性好
- 支持注释
- 易于手动编辑友好

## 限制与注意事项

1. **依赖 Google Chrome：
   - 必须安装系统 Chrome
   - 扫码登录必须使用真实 Chrome

2. **风控风险**：
   - 建议使用自然频率
   - 避免过于频繁发送
   - 消息内容需要自然

3. **登录有效期**：
   - 凭证有效期有限，可能需要定期重新扫码
   - 项目会自动刷新延长有效期
