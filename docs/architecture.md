## 核心工作流

### 1. 首次配置流程

```
用户点击「扫码登录」
    ↓
web/app.py → _run_setup_task()
    ↓
create_persistent_context() (打开真实Chrome)
    ↓
goto https://www.douyin.com/chat
    ↓
_find_qrcode() → 截图保存
    ↓
等待用户扫码 (轮询检查)
    ↓
is_logged_in() → 检查是否成功
    ↓
context.storage_state() → 保存 state.json
    ↓
返回成功状态
```

### 2. 日常运行流程

```
定时触发 / 手动点击「立即运行」
    ↓
_run_spark_task()
    ↓
get_browser() → 启动浏览器
    ↓
create_context() → 加载 state.json
    ↓
page.goto(chat_url)
    ↓
is_logged_in() → 验证登录
    ↓
find_and_send()
    ├─ 等待会话列表加载
    ├─ 遍历目标好友
    │   ├─ find_target_on_page()
    │   ├─ _do_send()
    │   │   ├─ build_message()
    │   │   ├─ 输入框输入
    │   │   └─ 按回车发送
    │   └─ 下一个好友
    └─ 滚动查找更多
    ↓
context.storage_state() → 刷新 state
    ↓
关闭浏览器
    ↓
返回运行结果
```

## 线程模型

### Web控制面板线程模型

```
主线程: Flask 服务
    ↓
├─→ 前台线程: 处理 HTTP 请求
├─→ 后台线程 1: APScheduler 调度器
└─→ 后台线程 2: 执行任务 (_run_spark_task / _run_setup_task)
```

### 任务执行线程

任务执行是阻塞式的，使用 Playwright 同步 API，在单个线程中顺序执行：

- 每个账号顺序处理
- 每个好友顺序发送
- 发送之间有适当延迟

## 安全性设计

1. **凭证本地存储**
   - state.json 保存在用户本地
   - 包含 cookies 和 localStorage
   - 不会上传到任何服务器

2. **反检测机制**
   - 使用真实 Chrome，而非自动化浏览器
   - 注入 stealth JS 隐藏自动化特征
   - 模拟真实用户操作行为

3. **配置隔离**
   - 用户数据与程序代码分离
   - 打包应用使用独立数据目录
